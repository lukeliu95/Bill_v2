## Gotchas
- **Python 必须**：bash grep+awk 管道会破坏字段格式（gsub 导致分隔符丢失）
- カテゴリID 是平台内部编号，不能用 categoryCodes 直接匹配
- 設立日很多企业为空，过滤时要保留空值记录
- 事業内容字段太短（通常一句话），单纯关键词匹配精度有上限
- 宽关键词（IT, システム, ソリューション）会捞进大量 IT 服务/コンサル公司 → 必须配合负向排除 + 评分
- 负向硬排除词：受託, 派遣, SES, 人材紹介, 請負, 常駐
- DB 员工数据可能过时（如 ABEJA: DB=98 → 实际300+）
